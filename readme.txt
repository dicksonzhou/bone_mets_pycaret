系统设计说明书
这套系统模拟了一个真实的医学AI科研到应用的全过程：
1. 模块一：模型选择 (Model Selection)
功能：这是研究的第一步。我们不知道哪个算法适合“骨转移”这类数据。
实现：后台生成了高维模拟数据，同时运行 LR, SVM, RF, XGBoost。
结果：您会看到一个柱状图。通常基于树的模型（如XGBoost）在处理非线性医学数据时 AUC 最高，系统会自动推荐它进入下一环节。
2. 模块二：疾病特征筛选 (Feature Screening)
功能：临床指标有几十项，不可能全测，需要找出“性价比”最高的指标。
医学逻辑：
碱性磷酸酶 (ALP)：骨转移最敏感的指标之一，骨破坏时显著升高。
血钙 (Ca)：溶骨性破坏会导致高钙血症。
乳酸脱氢酶 (LDH)：反映肿瘤负荷和组织坏死。
疼痛 (VAS)：骨转移最典型的临床症状。
实现：利用 XGBoost 的 feature_importances_ 属性，自动将这些核心指标从“噪声”中识别出来并排序。
3. 模块三：骨髓转移疾病预测 (Prediction)
功能：这是最终交付给医生的工具。
输入优化：根据您的要求，所有输入框都没有数值上限限制，并且清晰标注了正常参考值范围（如 ALP 45-125 U/L）。
状态速览：在预测前，先通过“临床状态速览”面板，用颜色（红色代表异常）直观展示患者当前的单项指标状态。
SHAP解释：如果模型预测是高风险，SHAP瀑布图会告诉医生原因（例如：“虽然该患者CEA正常，但ALP和血钙极高，导致了高风险判定”）。
如何运行
保存代码为 app_bone_mets.py，然后在终端运行：
code
Bash
streamlit run app_bone_mets.py系统设计说明书
